## 🎨 DESIGN ROADMAP

### Design Language

#### Brand Palette
| Color | Hex | Usage |
|---|---|---|
| Indigo | `#6366f1` | Primary brand color |
| Indigo Light | `#818cf8` | Hover states |
| Purple | `#a855f7` | Gradient accents |
| Deep Navy | `#0f0f1a` | Dark backgrounds |
| Surface | `#13132a` | Cards, panels |
| White | `#f0f0fa` | Primary text (dark mode) |
| Gold | `#fbbf24` | Highlights, badges |

#### Typography System
| Element | Font | Weight | Size |
|---|---|---|---|
| Display / Hero | Plus Jakarta Sans | 800 | 3rem–4.5rem |
| Headings | Inter | 700 | 1.5rem–2.25rem |
| Subheadings | Inter | 600 | 1.125rem–1.25rem |
| Body | Inter | 400 | 1rem |
| Labels | Inter | 500 | 0.75rem–0.875rem |
| Buttons | Inter | 600 | 0.875rem–1rem |
| Code | Source Code Pro | 400 | 0.875rem |

#### Design Tokens
- **Spacing**: 4px base unit (4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96, 128)
- **Border Radius**: 6, 12, 18, 24, 32, 9999px
- **Shadows**: 4 levels + glow variants
- **Animations**: Fast (150ms), Normal (250ms), Slow (400ms), Spring easing
- **Icons**: Lucide Icons (consistent, open-source)

#### Design Workflow
1. Design every screen in Figma before development
2. Extract design tokens to CSS custom properties
3. Build components in Fzyrk UI matching Figma specs
4. Design review before merging any UI PR
5. Maintain a living style guide
